Put spec/design content directly on the GitHub issue body, not as a link to a markdown file in the repo.

**Why:** Reviewers should be able to read the spec without navigating away from the issue. Linking to a file adds unnecessary friction.

**How to apply:** When creating issues that accompany a design spec, inline the full spec content in the issue body. The spec file in the repo can still exist for version control, but the issue should be self-contained.
